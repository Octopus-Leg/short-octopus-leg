# 제2부 | 논리적으로 사고를 정리하는 기술

## 제3장 중복, 누락, 혼재를 방지한다

### 1. MECE는 이야기의 중복, 누락, 혼재를 없애는 기술이다

전달자가 자신의 견해를 명확히 제시하고 상대를 논의의 영역으로 이끌기 위해서는 전체 집합을 중복이나 누락 없이 부분 집합으로 나누어 생각하는 MECE(Mutually Exclusive and Collectively Exhaustive) 사고방식이 필요함.

#### MECE란?

정보를 정리하는 세 가지 접근법을 비교하면 MECE의 가치를 이해할 수 있음.

- **나열식 접근법**: 생각나는 대로 정보를 열거하는 방식으로, 전체가 무엇인지 혹은 누락이 있는지 확인하기 어려움.
- **분류식 접근법**: 요일이나 시간 등 기계적인 규칙으로 분류하는 방식이나, 같은 내용이 중복되어 나오거나 정보의 성격이 기억에 남지 않는 단점이 있음.
- **MECE 접근법**: 전체 집합을 중복, 누락, 혼재가 없는 부분 집합들로 나누어 생각하는 방식으로, 상대가 전달자의 논리 구조를 쉽게 파악할 수 있게 도움.

#### 다양한 MECE 관점을 갖자

설명을 잘하는 사람은 한 가지 사항을 다양한 측면과 상황에서 설명함. MECE는 다음과 같이 크게 나눌 수 있음.

- 연령이나 성별처럼 완전히 요소를 분해할 수 있는 유형
- 이미 어떤 사항에 대해 MECE의 기준으로 나누어놓은 유형

#### 알아두면 편리한 MECE 프레임워크

- **3C(4C)**: 고객과 시장(Customer), 경쟁사(Competitor), 유통 채널(Channel), 자사(Company)를 통해 사업 환경 전체를 망라함.
- **4P**: 상품(Product), 가격(Price), 유통(Place), 촉진 전략(Promotion)을 통해 마케팅 전략의 일관성을 점검함.
- **흐름·단계**: 비즈니스 시스템 또는 밸류 딜리버리 시스템처럼 어떤 일이 일어날 때까지를 과정(Process)과 단계(Step)로 나누거나, 과거·현재·미래 또는 중기·장기와 같이 시간 축으로 분류함.
- **효율·효과**, **질·양**, **사실·판단** 등이 한 쌍의 개념으로 사용됨.

### 2. 그룹핑은 MECE를 활용한 정보 정리 작업이다

그룹핑(Grouping)은 흩어져 있는 수많은 정보를 MECE 기준에 따라 의미 있는 그룹으로 분류하는 과정임.

- 정보를 모두 꺼내놓은 뒤, 과제 답변에 적절한 MECE 기준을 찾아 분류함.
- 각 그룹에 적절한 제목을 붙이고, 이 제목들을 모았을 때 과제에 대한 전체 답변이 되는지 재확인함.
- 그룹핑의 목적은 전체 상황을 알기 쉽게 보여줌으로써 상대가 결론을 이해하게 만드는 데 있음.

#### 중복, 누락, 혼재가 없는 부분 집합을 만들어라

- 두 가지 이상의 그룹에 동시에 속하거나 어느 그룹에도 속하지 않는 정보가 있다면 해당 MECE 기준이 잘못된 것임.
- 수집한 정보를 나누기만 할 게 아니라 나눈 그룹 사이에 서로 MECE 관계가 성립되어야 함.

## 제4장 이야기의 비약을 없앤다

전달하려는 결론과 근거, 혹은 결론과 방법 사이의 맥락을 상대가 쉽게 이해하도록 만들기 위해 필요한 기술임.

### 1. So What?/Why So?는 이야기의 비약을 없애는 기술이다

- **So What?(그래서?)**: 수집한 정보로부터 "결국 어떻다는 것인지" 과제 답변에 맞는 핵심 요점을 추출하는 작업임.
- **Why So?(왜 그렇지?)**: So What?한 내용이 타당한지 자료를 통해 검증하는 작업으로, "왜 그렇게 말할 수 있는지"를 설명함.
- 결론(X)에 대해 Why So?라고 물었을 때 근거(A, B, C)가 답이 되어야 하며, 반대로 근거들을 So What? 했을 때 결론이 도출되어야 비약이 사라짐.

#### So What?/Why So?하는 습관을 들여라

- "그렇다면 이 자료에서 무엇을 알 수 있는가?", "결국 이 이야기에서 중요한 것은 무엇인가?" 생각하는 습관을 들여야 함.
- 이해력이 빠르다고 평가받는 사람들은 대부분 자료를 읽거나 상대의 말을 들으면서 결론과 핵심을 빠르고 정확하게 추출하는 So What? 능력이 뛰어남.

### 2. 두 종류의 So What?/Why So?

#### 관찰의 So What?/Why So?

- 제시된 사실(전체 집합)을 통해서 할 수 있는 이야기를 요약하는 작업임.
- "결론적으로 이 자료에서 무엇을 알 수 있는가"를 상대가 똑같이 이해할 수 있도록 명시해야 함.

### 사고방식과 해답 사례

1. 도표의 주제, 곧 전체 집합이 무엇인지 확인한다
2. 각각의 사실을 관찰하고 Why So?로 확인하면서 So What?을 생각한다
3. 도표의 주제를 근거로 각 관찰의 So What?을 요약한다

#### 통찰의 So What?/Why So?

- 어떤 상황을 나타내는 여러 자료에서 일정한 규칙과 법칙성이라든지, 회사가 취할 대책이나 회사에 미치는 영향을 생각하게 하는 어떤 정보로부터 다른 정보를 이끌어내는 작업임.
- "결국 어떤 상황인지", "결국 어떤 행동을 해야 하는지"를 추출하는 관찰과 관찰한 정보에서 업계의 구조나 회사가 해야 할 일 등 종류가 다른 사고와 판단을 이끌어내는 통찰은 차이가 있음.

### 사고방식과 해답 사례

1. 과제를 확인한다
2. 각각의 사실에 대해 우선 관찰의 So What?/Why So?를 한다
3. 관찰의 So What?/Why So?를 주시하고 각 회사의 상황에서 일정한 법칙성을 이끌어낸다
4. Why So?로 검증한다

#### 통찰의 So What?은 관찰의 So What?이 전제돼야 한다

- 상대의 Why So?에 답변할 수 있어야 함.
- Why So?의 답변이 "이건 어디까지나 제 의견입니다만", "만약 이런 일이 있다면", 세상 사람들의 80%가 상식적으로 생각해보지 않았을 것 같은 내용이라면 타인을 설득할 수 없음.
- 통찰의 So What?은 반드시 정확한 사실 관찰인 관찰의 So What?이 전제되어야 타당성을 얻을 수 있음.
