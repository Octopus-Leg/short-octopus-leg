# 1장 | 개발자가 알아야 할 글쓰기 기본

## 01 문장과 단락을 구조화하는 법
- **핵심 중심의 문장 구성**
    - 복잡한 구조 대신 **'핵심 + 부가 설명'** 의 형태로 쓰면 의미 전달이 명확함.
    - 예: `입력 데이터는 3차원 벡터다(핵심). 색상 RGB 값을 각각 사용하기 때문이다(부가 설명).`
- **서술식** vs **개조식** vs **도식**
  1. **서술식**: `~다`로 끝나는 완전한 문장 구조임. 줄거리나 이야기를 전달할 때 적합함.
  ```text
  이번 특강 주제는 개발자의 글쓰기이고, 2월 28일 오후 6시에 문어 연회장에서 진행됩니다.
  ```
  2. **개조식**: 신문의 헤드라인을 쓰거나 어떤 사항을 나열할 때 사용하는 구조임. 명사나 용언의 명사형으로 끝남.
  ```text
  - 특강 주제: 개발자의 글쓰기
  - 일시: 2월 28일 오후 6시
  - 장소: 문어 연회장
  ```
  3. **도식**: 행과 열로 이뤄진 표임. 행렬에 글만 있으면 표고, 막대 같은 그림이 있으면 도표임. 중복과 누락을 방지하고 항목 간의 차이를 더 분명하게 드러낼 수 있음.

- **개조식 작성을 위한 글머리 기호 활용**
  - **설명**: 내용을 구체적으로 설명하거나 나열할 때 `◼`, `◦`, `-`, `▪`, `✔` 등을 사용함.(하위 요소일수록 작아지고 들여쓰기를 적용함).
  - **묘사**: 그림의 요소나 영역을 표시할 때 `①`, `②`, `③`, `ⓐ`, `ⓑ`, `ⓒ` 등을 사용함.
  - **논증**: 논리적 인과관계를 나타낼 때 `➔`, `➜`, `∴`, `∵` 등을 사용함.
  - **서사**: 순서나 단계를 나타낼 때 `1`, `2`, `3`, `가`, `나`, `다` 등을 사용함.

- 들여쓰기를 할 때는 스페이스바 연타 대신 **탭과 스타일 설정**을 사용하여 일관성을 유지함.
- 문서의 위계를 명확히 드러내어 독자가 전체 구조를 쉽게 파악할 수 있도록 함.
  

## 02 쉽게 쓰는 띄어쓰기와 문장 부호
- 슬라이드, 마크다운 플랫폼, 블로그 글에도 띄어쓰기와 맞춤법을 잘 지켜서 글을 써야 함.

## 03 영어 단어 선택과 외래어 표기법
- **맥락에 맞는 대응어 선택**
  - 일반적인 `head`의 반대는 `tail/foot`이지만, HTML 환경에서는 `header`와 `footer`로 짝을 맞추는 것이 자연스러움.
- **상태 관련 비슷한 단어**
  - `stop`: 잠시 중단(언제든 재시작 가능함)
  - `end`: 완전 중단(재시작 가능성 없음)
  - `finish`: 완료(목표를 달성하여 끝냄)
  - `pause`: 일시 정지(매우 짧은 중단함)
  - `suspend`: 보류(다음 단계로의 진행을 멈춤)
  - `hold`: 의도적 대기(특정한 목적을 위해 중단함)
- **강제성**: `should`(권장 사항) vs `must`(필수 요구 사항)
- **수정**: `change`(단순 변경), `modify`(오류 교정), `revise`(정보 추가 및 개정)

- **일관성과 개연성 유지**
  - 정확한 단어 선택도 중요하지만, 문서 전체에서 같은 의미를 가진 단어를 **일관되게** 사용하는 것이 더 중요함.
  - 외래어 표기법에 얽매이기보다, 한 문서나 조직 내에서 하나로 통일하여 사용하면 됨.(예: 어노테이션 vs 애노테이션)
